CMU Dialog systems

The following is an example dialog from the CMU Darpa Communicator. Communcator is a automated telephone based dialog systems for booking flight information. The following dialog is an actual dialog, but the agent side has been replaced with synthesizer output using the Limited Domain Synthesis techniques as described in the FestVox project

When the limited domain synthesizer fails, it defaults to using the diphone synthsizer (which is built from the same speaker), though doesn't happen in this example. Note although selections in the limited doamin synthesizer do use then word name as an index this is different from simple word and phrase concatenation as the join will often ocurr within word, as that's is determined to be the best join point. If you listen carefully there are occasionally errors in the synthesis, the word "air" in the final email address is very short, and some joins are not very smooth. However the quality of synthesis is far superior than standard text to speech.

The voices was constructed from 630 in-domain utterances selected by looking at the most frequent utterances made by the communciator over a 3 month period, plus others to ensure word coverage of the domain.
